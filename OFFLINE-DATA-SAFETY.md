# 离线数据安全说明

## 核心目标

即使网络不稳定，用户在录入过程中的数据也不应丢失。

## 当前正式流程

统一从：

- `/analysis/new`

进入诊断流程。

不再使用旧入口：

- `/analysis/new/mobile`

## 典型场景

### 场景 1：正常流程

1. 打开 `/analysis/new`
2. 选择项目与对象
3. 填写训练或比赛信息
4. 保存后查看诊断结果

### 场景 2：中途断网

1. 在 `/analysis/new` 中录入到一半
2. 网络中断
3. 页面仍以本地状态继续保留输入
4. 网络恢复后再继续提交

### 场景 3：关闭后回到页面

1. 用户中断录入
2. 稍后重新进入 `/analysis/new`
3. 由本地状态恢复最近的录入上下文

## 原则

- 离线能力服务于统一工作台
- 不再用单独移动端路由承载离线逻辑
- 任何新的离线说明都应围绕 `/analysis/new`
