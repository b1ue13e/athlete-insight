## 第一层：体验层优化

### 1.1 语音录入系统

**核心价值**: 将 90 秒快录变成 30 秒，满手是汗也能操作

**实现文件**: `components/voice-input.tsx`

**使用方式**:
```tsx
import { VoiceInput, VoiceBatchInput } from "@/components/voice-input"

<VoiceInput onResult={(events) => { /* 处理解析结果 */ }} />
<VoiceBatchInput onComplete={({ events, summary }) => { /* 批量处理 */ }} />
```

**支持的语音指令**:
- "主攻手张三，四号位重扣得分，二传给得到位"
- "发球失误一个，跳发下网"
- "拦网得分，单人拦住对方接应"
- "一传到位，直接组织快攻"

---

### 1.2 高光雷达图/球星卡分享

**核心价值**: 从"看数据"到"秀实力"，让分享成为社交货币

**实现文件**: `components/share-cards.tsx`

**三种卡片风格**:
1. **雷达图风格**: 四维能力雷达图，击败同位置球员百分比
2. **球星卡风格**: 稀有度标签 (LEGENDARY/ELITE/RARE)，收藏级视觉
3. **极简风格**: 纯文字+表情，一键复制分享文案

## 第二层：算法层优化

### 2.1 xP (期望得分) 模型

**核心价值**: 同样的得分，面对三人拦网强攻 vs 探头球，含金量完全不同

**实现文件**: `lib/xp-model.ts`

**攻击情境权重**:
- `open_net`: 基础0.65，难度1.0
- `double_block`: 基础0.45，难度1.3
- `triple_block`: 基础0.25，难度1.8
- `chaos_ball`: 基础0.20，难度1.6
- `tip_over`: 基础0.85，难度0.6

---

### 2.2 置信区间评分系统

**核心价值**: 不是"本场得分：82"，而是"综合评分：78-86（高置信度）"

**实现文件**: `lib/confidence-interval.ts`

**质量评级**:
- 高置信度 (≥85%): 数据充分，评分可信
- 中等置信度 (70-84%): 基本充分，存在不确定性
- 低置信度 (50-69%): 数据不足，仅供参考
- 数据不足 (<50%): 无法做出可靠评估

---

### 2.3 贝叶斯个体基线更新

**核心价值**: 用贝叶斯更新动态计算真实能力是否发生实质性漂移

**实现文件**: `lib/bayesian-baseline.ts`

**核心算法**:
- 先验：历史表现
- 似然：本场表现
- 后验：更新后的能力估计
- 漂移检测：基于t检验的统计显著性判断

---

## 第三层：AI应用层深化

### 3.1 结构化Prompt与因果推理

**核心价值**: 让 AI 成为数据分析师而不是文案员

**实现文件**: `lib/ai-analyst.ts`

**设计要点**:
- Payload 包含完整比赛事件流序列
- 要求 AI 识别因果模式（不只是描述）
- 输出可操作性建议

---

### 3.2 训练动作库向量化匹配

**核心价值**: 不让 AI 生成训练动作（容易幻觉），而是检索匹配专业动作库

**实现文件**: `lib/training-vector-store.ts`

**预置动作**: 发球、一传、进攻、拦网、防守、关键分、体能等15+专业训练动作

**匹配算法**: 余弦相似度计算问题向量与动作向量的匹配度

---

## 第四层：未来拓展

### 4.1 跑步高阶指标设计

**实现文件**: `lib/running-advanced-metrics.ts`

**高阶指标**:
1. **心率漂移 (HR Decoupling)**: 评估有氧耐力底子
2. **步频与疲劳度关联**: 检测疲劳时的技术变形
3. **后半程掉速曲线**: 配速衰减速率分析
4. **乳酸阈值边缘表现**: 类比排球关键分表现
