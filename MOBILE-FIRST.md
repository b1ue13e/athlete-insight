# 移动端录入说明

## 现状

项目不再保留独立的移动端录入路由。

以下旧入口已废弃：

- `/analysis/new/mobile`

移动端用户现在也统一从：

- `/analysis/new`

进入同一套诊断工作台。

## 设计原则

- 同一套工作台壳层
- 同一套导航与入口
- 同一套录入流程
- 移动端通过响应式布局适配，而不是单独维护第二套页面

## 当前建议

如果需求是：

- 场边快速录入
- 弱网 / 断网下继续填写
- 手机上继续补录

也应直接使用：

- `/analysis/new`

而不是新增新的分支入口。

## 禁止回归

后续不要再引入：

- `/analysis/new/mobile`
- “移动端专用诊断入口”
- 基于平台拆分的第二套录入页面

如需调整，请先检查：

- `docs/redirects-checklist.md`
