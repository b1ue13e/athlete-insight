## 轻登录方案

### 选定：Magic Link（最轻）

**用户体验**:
1. 输入邮箱
2. 收到邮件，点击链接
3. 自动登录

**不需要**:
- 密码
- 注册表单
- 邮箱验证
- 找回密码

**技术实现**:
- 暂时用本地存储模拟（测试阶段）
- 后续接 Supabase Auth（生产阶段）

## 本地/云端数据合并规则

**优先级**（避免数据冲突）:
1. 每条记录有稳定本地 ID（UUID）
2. 登录后先"本地未同步数据上推"
3. 云端已存在的记录（按ID查重）不重复插入
4. 草稿和正式提交状态严格区分
5. 冲突时以"最新更新时间"为准

**数据流程**:
```
离线录入 → IndexedDB → 联网 → 检查登录
                               ↓
                     已登录 → 同步到云端
                     未登录 → 暂存本地，登录后上推
```

## 内部数据看板

### 每周必须看的指标

| 指标 | 目标线 | 低于线怎么办 |
|------|--------|-------------|
| 首次录入完成率 | > 70% | 检查哪一步退出最多 |
| 平均快录时长 | < 180秒 | 简化步骤，减少输入 |
| 第二次使用率 | > 50% | 加强提醒，优化回访 |
| 分享率 | > 30% | 降低分享门槛 |
| 补录率 | > 20% | 强调补录价值 |
| 处方反馈率 | > 25% | 简化反馈流程 |
| 7日留存 | > 40% | 分析流失原因 |

### 看板页面
路径：`/admin/dashboard`（密码保护）

显示：
- 本周活跃用户数
- 总录入场次
- 快录/补录占比
- 漏斗图（开始→完成→分享→反馈）
- 退出点分布
- 平均录入时长趋势

## 10用户测试准备

### 用户分层
| 类型 | 人数 | 特征 |
|------|------|------|
| 普通球员 | 5 | 偶尔录，不太规律 |
| 自律用户 | 3 | 愿意连续录多场 |
| 核心用户 | 2 | 队长/教练/懂球 |

### 测试群
- 微信群（收反馈、截图）
- 每周一次简短语音/文字回访

### 反馈表（每场录完30秒）
1. 这次录入顺不顺？（1-5分）
2. 哪一步最烦？（选择题）
3. 报告准不准？（1-5分）
4. 建议有没有用？（1-5分）
5. 下次还愿不愿意录？（是/否）

### 每周内部板
```

================

总场次: 23
```
