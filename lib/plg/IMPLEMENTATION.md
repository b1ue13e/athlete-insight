# 产品驱动增长 (PLG) 三层实现总结

## 已完成内容

### ✅ 第一层：剧本式引导 (The Guided Tour)

**文件**: `components/onboarding/guided-tour.tsx`

**实现**: 使用 driver.js 创建三步剧本引导

**剧本拆解**:

```
第一幕 (看痛点): 
  → 高亮雷达图防守短板
  → Tooltip: "看这里，虽然张三得分高，但防守端的凹陷是第三局被逆转的致命伤"

第二幕 (看 AI 处方):
  → 自动滚动到处方卡片
  → Tooltip: "不用你自己想对策，AI 已经开出了针对性的训练处方"

第三幕 (收网钩子):
  → 高亮"导入数据"按钮
  → Tooltip: "花 90 秒，看看你自己的球队里藏着哪些致命漏洞"
```

**核心洞察**:
> "用户很懒，如果不告诉他们看哪里，他们只会觉得花里胡哨"

---

### ✅ 第二层：柔性门槛转化 (Soft Gating)

**文件**: `components/conversion/soft-gate-modal.tsx`

**交互流**:

```
1. 用户点击上传
   ↓
2. 显示炫酷进度条 (制造期待感)
   - 正在解析数据... (15%)
   - 异常值清洗中... (35%)
   - 计算夏普比率... (55%)
   - 构建协方差矩阵... (75%)
   - 生成 AI 处方... (90%)
   - 报告已就绪！ (99%)
   ↓
3. 弹出高转化率 Modal
   - 背景：毛玻璃效果的真实报告预览
   - 文案："报告已就绪！发现了 X 处风险"
   - 行动：邮箱/Google 登录解锁
```

**金句**:
> "最高级的逼单，不是把门锁上，而是把门开一条缝，让他看到里面的金山，然后告诉他进门只需要签个字。"

---

### ✅ 第三层：健壮 Excel 解析 (Robust Parser)

**文件**: `components/import/excel-parser.tsx`

**工程解法**: xlsx + Zod 防御性编程

**三层架构**:

```
1. 读取层 (xlsx)
   - 读取 Sheet1
   - 提取表头和预览数据
   
2. 映射层 (半自动 UI)
   - 左边：教练的原始表头
   - 右边：下拉框选择标准字段
   - 自动建议：根据字段相似度预填充
   
3. 校验层 (Zod Schema)
   - 类型强制转换 (coerce.number())
   - 必填字段验证
   - 无效数据高亮显示
```

**字段映射建议**:
```typescript
const FIELD_SUGGESTIONS = {
  name: ['姓名', '名字', 'name', 'player'],
  attackKills: ['扣球得分', '进攻得分', 'attack_kills', 'kills'],
  // ...
}
```

**Zod 验证示例**:
```typescript
const PlayerDataSchema = z.object({
  name: z.string().min(1, '姓名不能为空'),
  attackKills: z.coerce.number().min(0).default(0),
  // ...
});

// 验证失败时高亮显示
if (!parseResult.success) {
```

**功能整合**:

## 使用方式

**访问路径**: `/plg-demo`

**体验流程**:
1. 首次进入 → 自动触发三步引导
2. 点击"上传" → 观看进度条动画
3. 走到 99% → 弹出转化模态框
4. 或点击"高级导入" → 体验字段映射
