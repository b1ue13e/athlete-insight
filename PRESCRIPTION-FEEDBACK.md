# 处方执行回访系统

## 功能概述

让产品知道"建议有没有用"，从"会提建议"升级为"会追踪建议效果"。

---

## 核心功能

### 反馈收集
- **执行情况**: 已完成 / 部分完成 / 未执行
- **自我感受**: 有改善 / 无明显变化 / 更差
- **备注**: 可选文字记录

### 效果统计
- **执行率**: (已完成 + 部分完成) / 总次数
- **有效率**: 改善次数 / 执行次数
- **趋势判断**: 
  - 持续改善 (近2次都改善)
  - 效果稳定
  - 近期回落

### 智能建议
根据历史反馈自动生成建议：
- "该处方尚未执行，建议优先完成"
- "多次执行效果不明显，建议调整训练形式"
- "该处方对你效果显著，建议继续保持"

### 数据结构

```typescript
interface PrescriptionFeedback {
  id: string
  reportId: string
  athleteId: string
  prescriptionId: string
  executionStatus: "completed" | "partial" | "not_done"
  perceivedEffect: "improved" | "no_change" | "worsened"
  recordedAt: string
}

interface PrescriptionEffectSummary {
  totalAssigned: number
  completedCount: number
  partialCount: number
  effectivenessRate: number  // 有效率
  trend: "improving" | "stable" | "declining"
}
```

## 产品价值

1. **闭环验证** - 知道建议是否有效
2. **个性化优化** - 基于个人反馈调整建议
3. **增加粘性** - 用户会回来"打卡"训练
4. **数据积累** - 什么样的处方对什么人有效果
